= Hager ECR380D Meter

Three-phase electricity meter for energy measurement via Modbus protocol.

== Overview

This bundle provides integration with the Hager ECR380D three-phase electricity meter, designed for precise energy measurement in commercial and industrial installations. The component communicates with the meter through the Modbus TCP protocol, enabling real-time energy monitoring, power measurement, and comprehensive data collection in OpenEMS.

The Hager ECR380D supports measurement of grid consumption, production, or consumption energy depending on configuration. It provides detailed per-phase and cumulative measurements for complete energy monitoring.

== Supported Devices

* https://www.hager.com/[Hager ECR380D Meter icon:external-link[]]

== Components

This bundle implements the following OpenEMS Component:

=== Hager ECR380D Meter

*Name*: Meter Hager ECR380D

*Factory-PID*: `Meter.Hager.ECR380D`

.Implemented Natures/Interfaces
* HagerEcr380dMeter
* ElectricityMeter
* ModbusComponent
* OpenemsComponent
* ModbusSlave

*Description*: Three-phase electricity meter for energy measurement via Modbus protocol.

*Configuration*:
.Configuration Parameters
* `id` (String): Component ID for unique identification (default: "meter0")
* `alias` (String): Human-readable alias for the component
* `enabled` (Boolean): Enable/disable this component (default: true)
* `modbus_id` (String): ID of Modbus bridge for communication (default: "modbus0")
* `modbusUnitId` (Integer): Modbus Unit ID of the meter device (required)
* `type` (Enum): Meter type - GRID, PRODUCTION, or CONSUMPTION (default: PRODUCTION)

== Prerequisites / Dependencies

* Modbus bridge configured and running (e.g., `io.openems.edge.bridge.modbus`)
* Modbus TCP connectivity to the ECR380D meter
* Correct Modbus Unit ID configured on the meter
* Proper three-phase power wiring

== Features

* Three-phase power measurement with individual phase data
* Reactive and apparent power measurement
* Power factor measurement per phase and overall
* Energy counters (active and reactive)
* Frequency measurement
* Comprehensive meter data logging

== Known Limitations

* Meter type must be pre-configured - cannot be changed dynamically
* Modbus communication is required - no alternative protocols supported
* Modbus Unit ID varies by installation - verify correct address on meter
* Modbus response time depends on network connectivity and meter load
