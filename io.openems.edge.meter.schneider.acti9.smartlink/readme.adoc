= Schneider Acti9 Smartlink Meter

Three-phase electricity meter for energy measurement via Modbus protocol.

== Overview

This bundle provides integration with Schneider Acti9 Smartlink meters, three-phase electricity meters designed for energy management in commercial and industrial installations. The component communicates with the meter through the Modbus TCP protocol, enabling real-time energy monitoring, power measurement, and comprehensive data collection in OpenEMS.

Schneider Acti9 Smartlink meters are part of Schneider Electric's distribution management system for reliable energy measurement and monitoring.

== Supported Devices

* https://www.schneider-electric.com/[Schneider Acti9 Smartlink Meter icon:external-link[]]

== Components

This bundle implements the following OpenEMS Component:

=== Schneider Acti9 Smartlink Meter

*Name*: Meter Schneider Acti9 Smartlink

*Factory-PID*: `Meter.Schneider.Acti9.Smartlink`

.Implemented Natures/Interfaces
* MeterSchneiderActi9Smartlink
* ElectricityMeter
* ModbusComponent
* OpenemsComponent

*Description*: Three-phase electricity meter for energy measurement via Modbus protocol.

*Configuration*:
.Configuration Parameters
* `id` (String): Component ID for unique identification (default: "meter0")
* `alias` (String): Human-readable alias for the component
* `enabled` (Boolean): Enable/disable this component (default: true)
* `modbus_id` (String): ID of Modbus bridge for communication (default: "modbus0")
* `modbusUnitId` (Integer): Modbus Unit ID of the meter device (default: 1)
* `type` (Enum): Meter type - GRID, PRODUCTION, or CONSUMPTION (default: PRODUCTION)
* `invert` (Boolean): Invert all power and current values (default: false)

== Prerequisites / Dependencies

* Modbus TCP bridge configured and running with connection to the correct meter IP address (e.g., `io.openems.edge.bridge.modbus`)
* Modbus TCP connectivity to the Schneider Acti9 Smartlink meter
* Correct Modbus Unit ID configured on the meter (default: 1)
* Proper three-phase power wiring

== Configuration Setup

To configure this meter:

. First, add a Modbus TCP bridge component connecting to the meter's IP address
. Configure this meter component to use the configured Modbus bridge
. Set the Modbus Unit ID (default: 1)
. Set the meter type (GRID, PRODUCTION, or CONSUMPTION)
. Enable the component

== Features

* Three-phase power measurement with individual phase data
* Reactive and apparent power measurement
* Power factor measurement per phase and overall
* Energy counters (cumulative consumption/production)
* Frequency measurement
* Comprehensive meter data logging

== Known Limitations

* Meter type must be pre-configured - cannot be changed dynamically
* Modbus communication is required - no alternative protocols supported
* Invert option affects all power and current measurements
* Default Modbus Unit ID is 1 - verify correct address on your meter
