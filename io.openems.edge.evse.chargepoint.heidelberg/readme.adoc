= Heidelberg Connect Charging Station

Three-phase or single-phase AC charging station with Modbus integration.

== Overview

This bundle provides integration with the Heidelberg Connect electric vehicle charging station, a flexible AC charging solution supporting both single-phase and three-phase configurations. The station communicates via Modbus for integration with energy management systems, offering phase-switching capability for dynamic load balancing.

Heidelberg Connect is suitable for commercial charging networks, workplace charging, and residential installations requiring flexible single/three-phase charging.

== Supported Devices

.Heidelberg Connect Series
* **Type**: AC charging station (single-phase or three-phase configurable)
* **Typical Power**: 3.7 kW (1-phase) to 22 kW (3-phase)
* **Protocol**: Modbus TCP/RTU
* **Communication**: Modbus over Ethernet or serial
* **Typical Use**: Commercial charging, workplace charging, residential
* **Connector**: Type 2 (Mennekes) standard
* **Phase Switching**: Optional dynamic phase adjustment
* **Documentation**: https://www.heidelbergelectric.com/[Heidelberg Electric icon:external-link[]]

== Components

This bundle implements the following OpenEMS Components:

=== <<_heidelberg_connect,EVSE Charge-Point Heidelberg Connect>>

*Name*: EVSE Charge-Point Heidelberg Connect

*Factory-PID*: `Evse.ChargePoint.Heidelberg.Connect`

.Implemented Natures/Interfaces
* EvseChargePointHeidelbergConnect
* ModbusComponent
* OpenemsComponent
* TimedataProvider
* EvseChargePoint
* EventHandler
* ElectricityMeter

*Description*: Modbus interface for Heidelberg Connect charging station. Provides charging point functionality with real-time power measurement and status monitoring. Supports both single-phase and three-phase configurations with optional dynamic phase switching for load optimization.

*Configuration*:
.Configuration Parameters
* `id` (String): Component ID for unique identification (default: "evseChargePoint0")
* `alias` (String): Human-readable alias for the component
* `enabled` (Boolean): Enable/disable this component (default: true)
* `readOnly` (Boolean): Read-only mode (monitoring only) vs managed control (required)
* `debugMode` (Boolean): Enable debug logging (default: false)
* `wiring` (SingleOrThreePhase): Hardware wiring configuration - SINGLE_PHASE or THREE_PHASE (required)
* `phaseSwitching` (PhaseSwitching): Phase switching option - FORCE_THREE_PHASE, FORCE_SINGLE_PHASE, or AUTO (default: FORCE_THREE_PHASE)
* `phaseRotation` (PhaseRotation): Phase order - L1_L2_L3 or rotated configurations (default: L1_L2_L3)
* `modbus_id` (String): ID of the Modbus bridge to use (default: "modbus0")
* `modbusUnitId` (Integer): Modbus Unit-ID of the device (default: 1)

.Example Configuration
For a three-phase Heidelberg Connect with phase switching:
* `id`: "evseChargePoint0"
* `alias`: "Heidelberg Connect Charger"
* `modbus_id`: "modbus0"
* `modbusUnitId`: 1
* `wiring`: "THREE_PHASE"
* `phaseSwitching`: "AUTO"
* `phaseRotation`: "L1_L2_L3"
* `readOnly`: false

== Prerequisites / Dependencies

* Modbus bridge component must be configured and running
* Heidelberg Connect charging station must be physically installed and powered on
* Modbus TCP or RTU communication working between system and charger
* Device must be properly addressed on Modbus bus
* Network connectivity from OpenEMS to Modbus bridge
* Wiring must match configured single-phase or three-phase setting
* Phase rotation must correspond to actual electrical installation
* Device firmware should be current

== Features

* Flexible single-phase or three-phase charging (3.7-22 kW)
* Modbus protocol for standard integration
* Real-time power consumption monitoring
* Dynamic charging current control
* Optional phase switching for load optimization
* Support for demand response and load balancing
* Integration with energy management systems
* Read-only or managed control modes
* Phase rotation detection and configuration
* Time-of-use charging capability
* Comprehensive status monitoring

== Known Limitations

* Requires active Modbus connection - no local operation
* Modbus timeouts will prevent charging operations
* Wiring configuration must match actual installation (single vs three-phase)
* Phase rotation must be manually configured to match installation
* Phase switching may cause brief charging interruption during transitions
* Charging current adjustment limited by hardware capabilities
* Single-phase and three-phase cannot be dynamically switched without reconfiguration

== Phase Switching Notes

The Heidelberg Connect supports dynamic phase switching to optimize power consumption:

.Phase Switching Options
* `FORCE_THREE_PHASE`: Always use three-phase (maximum power)
* `FORCE_SINGLE_PHASE`: Always use single-phase (lower power, less grid load)
* `AUTO`: Automatically switch based on grid conditions and energy availability

Phase switching changes may temporarily interrupt charging - plan transitions during acceptable times.
